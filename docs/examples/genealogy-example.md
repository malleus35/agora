# Genealogy / Assumption Audit Example

## Prompt

```bash
/assumption-audit "Audit the assumptions behind calling Agora a multi-agent debate harness rather than a supervised judgment overlay"
```

## Example output

```markdown
## Assumption Audit

### Explicit Assumptions
- Users understand the value of debate immediately.
- Philosopher roles are the clearest entrypoint for adoption.
- More visible multi-agent behavior makes the product feel more distinctive.

### Hidden Assumptions
- Users are willing to switch workflows if the worldview is interesting enough.
- Dramatic orchestration helps trust more than concrete artifacts do.
- The word "debate" sounds rigorous rather than theatrical.

### Inherited Categories
- "agent framework" is being treated as the natural category.
- "multi-agent" is being used as a prestige label even when the user really wants better supervision.
- The product is inheriting evaluation standards from orchestration-heavy tools instead of judgment tools.

### Value Conflicts
- distinctiveness vs adoption
- philosophical worldview vs low-friction utility
- visible orchestration vs durable artifacts

### What if the Opposite Were True?
- What if users do not want more visible agent theater?
- What if the product becomes stronger by hiding the philosophical machinery behind small commands and reliable artifacts?
- What if the real differentiation is not multi-agent debate, but supervised judgment under uncertainty?

### Revaluation
- Stop asking, "How do we make the debate more impressive?"
- Ask instead, "How do we make clarification, dissent, doubt, and approval usable inside existing agent workflows?"
```

## Why this matters

This example shows Nietzsche's role in v2.
The goal is not branding flair first.
The goal is exposing the inherited category that is distorting the product surface.
