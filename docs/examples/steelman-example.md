# Steelman Example

## Prompt

```bash
/steelman "Restate the strongest case for delaying a host-agnostic Agora core and remaining plugin-first for now"
```

## Example output

```markdown
## Steelman

### Target Position
- Stay plugin-first in v2 instead of splitting Agora into a separate host-agnostic core now.

### Strongest-Form Restatement
- Agora's current bottleneck is not technical modularity but adoption clarity.
- A repo split would optimize an abstraction that has not yet proven repeated real-world use.
- Keeping one branded repo preserves conceptual coherence while the team validates whether the core judgment workflows are genuinely reusable across hosts.
- Premature separation would increase maintenance, create naming confusion, and risk producing two weak products instead of one legible one.

### What This Position Protects
- adoption friction
- brand coherence
- execution focus
- proof-before-abstraction discipline

### Fair Critique Boundary
- A valid critique must show either:
  - that cross-host reuse is already frequent enough to justify separation now, or
  - that keeping one repo is actively blocking host support and contributor understanding.
- An invalid critique would merely say, "modularity is always better," without addressing timing, adoption stage, or maintenance cost.
```

## Why this matters

Steelman is useful when a team is criticizing too quickly.
It preserves disagreement quality before compare-options, minority-report, or court-review.
