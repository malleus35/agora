# Dialectic Example

## Prompt

```bash
/dialectic "Should Agora prioritize minimal overlay adoption or deeper philosophical differentiation in v2?"
```

## Example output

```markdown
## Dialectic Result

### Thesis
- Prioritize minimal overlay adoption.
- The first obligation is to make Agora easy to try inside existing host workflows.

### Antithesis
- Prioritize deeper philosophical differentiation.
- Without a stronger worldview surface, Agora risks becoming a generic skill bundle with unusual names.

### Preserved Strengths
- From thesis: low-friction adoption, quick time-to-value, less user intimidation.
- From antithesis: real differentiation, conceptual identity, stronger long-term brand.

### Synthesis
- Keep the surface area light at first use, but make the overlays explicit and deep once the user steps past the first command.
- In practice: small independent commands for entry, stronger dialectic/genealogy/court modes for repeated advanced use.

### Next Step
- Maintain `/clarify`, `/decide`, `/doubt` as the low-friction front door.
- Continue deepening philosophy-to-workflow docs and overlay-specific examples so differentiation is visible after first contact.
```

## Why this matters

This example shows where `/dialectic` is different from `/decide`.
The goal is not merely to choose one option, but to preserve two strong truths and find a higher-order path.
