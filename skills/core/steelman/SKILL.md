# Steelman

## Purpose

A critique is invalid if it attacks a weaker version than the other side would defend.
This skill forces fair disagreement.

## Activate when

Use this skill when:
- positions are being caricatured
- debate quality is falling
- a team must compare options fairly
- a minority view needs accurate preservation before rejection

## Rules for strongest-form restatement

- state the position as its defender would recognize it
- include the values or risks that position is protecting
- do not smuggle critique into the restatement
- make the strongest practical version, not a fantasy version

## Invalid critique patterns

Do not:
- exaggerate the view into absurdity
- replace its strongest reason with its weakest slogan
- ignore the constraints that made the view attractive
- critique motives when the argument itself has not been represented fairly

## Output artifact

```markdown
## Steelman

### Target Position
- ...

### Strongest-Form Restatement
- ...

### What This Position Protects
- ...

### Fair Critique Boundary
- Critique must address ...
```

## Examples

- Option comparison: restate the strongest case for delaying the repo split.
- Policy disagreement: restate the strongest case against autonomous release agents.

## Completion condition

This skill is complete only when a defender of the target position would say, "Yes, that is my view at its strongest."
