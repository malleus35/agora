# Assumption Audit

## Purpose

Expose the premises a team is relying on, especially the ones it does not notice.

## Activate when

Use this skill when:
- the team may be solving the wrong problem
- definitions feel inherited rather than examined
- a proposal sounds coherent but rests on fragile premises
- Nietzsche/genealogy mode is appropriate

## Required sections

- explicit assumptions
- hidden assumptions
- category critique
- value conflict

## Output artifact

```markdown
## Assumption Audit

### Explicit Assumptions
- ...

### Hidden Assumptions
- ...

### Inherited Categories
- ...

### Value Conflicts
- ...

### What if the Opposite Were True?
- ...
```

## Completion condition

Complete only when the team can see which assumptions are chosen and which were merely inherited.
